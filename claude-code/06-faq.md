## 基础使用

### Claude Code 和网页版 Claude 有什么区别？

**Claude Code** 是运行在终端里的命令行工具：
- 直接读取和修改你本地的文件
- 执行终端命令（git、npm 等）
- 通过 CLAUDE.md / Skill / Subagent 构建持久化的 AI 工作流

**网页版 Claude** 只能在对话框里交流，无法直接操作本地文件，也没有 Memory / Skill / Subagent 机制。

---

### 对话越来越长，Claude 开始"忘事"了怎么办？

上下文窗口有容量上限。对话过长时，Claude 会自动压缩早期内容，可能导致它遗忘之前讨论的细节。

几个应对方法：
- **开新对话**：复杂任务完成一个阶段后，开新对话继续，把关键结论写进 CLAUDE.md 永久保留
- **用 Subagent 隔离大任务**：让 Subagent 处理输出量大的步骤，主对话只接收结论
- **精简 CLAUDE.md**：控制在 200 行内，减少每次对话的固定消耗

---

### 如何让 Claude 在修改文件前先告诉我它要做什么？

在对话里直接说明：

```
在修改任何文件之前，先告诉我你打算改什么，等我确认后再动手
```

或者写入 CLAUDE.md，让这条规范在每次对话自动生效：

```markdown
## 操作规范
修改文件前必须先列出计划，经用户确认后再执行。
```

---

### 如何撤销 Claude 刚才对文件的修改？

如果项目在 Git 仓库里：

```bash
# 查看 Claude 改了哪些文件
git status
git diff

# 撤销所有未提交的修改（回到上次 commit 的状态）
git restore .

# 只撤销某个文件
git restore 文件路径
```

如果项目还没有用 Git 管理，建议从现在开始。没有 Git，Claude 改了文件就很难回退。

## CLAUDE.md

### CLAUDE.md 写了很多，但 Claude 好像没有遵守，为什么？

几个常见原因：

- **内容太长**：CLAUDE.md 超过 200 行后，Claude 对靠后内容的关注度会下降。把不常用的专项内容移到 Skill 里，按需触发。
- **指令不够具体**："保持简洁"不如"每段不超过 3 句话"，越具体越容易执行。
- **对话里的指令覆盖了规范**：如果当次对话里说了与 CLAUDE.md 矛盾的要求，Claude 会优先服从你当次的指令。

---

### 项目级和个人级 CLAUDE.md 有冲突怎么办？

两者都会被加载，内容叠加而非覆盖。冲突时 Claude 会尽量同时满足，但可能判断不准。

建议清晰划分职责：
- **项目级**（根目录 `CLAUDE.md`）：团队共识、产品背景、协作规范，提交 Git 共享
- **个人级**（`~/.claude/CLAUDE.md`）：个人偏好，不要与项目级内容重复

如果 Claude 行为混乱，可以直接问它：`你现在加载了哪些 CLAUDE.md？内容是什么？`

## Skills

### 如何查看当前加载了哪些 Skills？

在 Claude Code 对话中输入：

```
/help
```

或者直接问：

```
你现在有哪些可用的 skills？
```

---

### 新建了 Skill 文件，为什么 Claude 还是不能用？

Claude 在启动时扫描 skills 目录，运行过程中不会重新扫描。新建 Skill 文件夹后，必须**重启 Claude** 才能被识别和触发。Subagent 同理。

---

### Skill 触发了，但 Claude 没有按规范工作，怎么办？

几个排查方向：

- **description 不够精确**：Claude 靠 `description` 判断何时触发，描述模糊可能触发了但不是你期望的那个。检查 SKILL.md 的 description 是否准确描述了触发场景。
- **被对话指令覆盖**：当次对话里说了与 Skill 规范冲突的要求，Claude 会优先服从你的指令。
- **同名 Skill 被覆盖**：项目级（`.claude/skills/`）优先级高于个人级（`~/.claude/skills/`），同名时项目级生效。

可以直接问 Claude：`你现在触发了哪个 skill？把 SKILL.md 的内容告诉我。`

---

### 项目级和个人级同名 Skill，哪个会生效？

项目级（`.claude/skills/`）优先级高于个人级（`~/.claude/skills/`），同名时项目级覆盖个人级。

## Subagents

### Subagent 迟迟没有反应，是卡住了吗？

Subagent 在独立上下文中运行，复杂任务（如扫描整个代码库）正常需要几分钟。如果明显异常，按 `Ctrl+C` 中断，重新描述任务或拆成更小的步骤再试。

---

### Subagent 的结果不满意，怎么让它改？

Subagent 每次调用都是全新上下文，直接在主对话里反馈，Claude 会决定是否重新派遣。

如果想让 Subagent 在已有上下文的基础上继续调整，而不是从头开始，可以这样说：

```
继续刚才那个分析，但把结论部分改成表格格式
```

Claude 会 resume 之前的 Subagent，而不是重新启动一个。

---

### 为什么不直接让 Claude 在主对话里完成，要用 Subagent？

主要有两个原因：

1. **保护上下文**：扫描代码库、抓取网页、运行测试等任务会产生大量中间输出。放在主对话会快速占满上下文窗口，Subagent 只把结论返回，主对话保持干净。
2. **并行提速**：需要同时分析多个竞品、同时调研几个模块时，Subagent 可以并行运行，主对话顺序完成同样的事情要慢得多。

## 安全与权限

### Claude Code 会不会误删或误改我的文件？

Claude Code 执行文件操作前会展示操作内容并请求你的确认，默认情况下你可以在它动手前看到它打算做什么。

保险起见建议：
- 重要项目用 Git 管理，随时可以回滚（见上方"如何撤销修改"）
- 对不确定的操作，先让 Claude 列出计划再确认执行

---

### 我不小心 `rm` 了一个重要文件怎么办？

如果文件在 Git 仓库里：

```bash
# 查看文件的历史记录
git log -- 文件路径

# 恢复最近一次提交的版本
git checkout HEAD -- 文件路径
```

如果不在 Git 仓库里，可以尝试从 Time Machine 恢复，或联系 IT。这也是建议用 Git 管理所有重要文件的原因之一。
